1) Sempre que um componente Angular faz uso de outro componente
Angular através de um referenciamento, dizemos que o componente
referenciado é um componente filho do componente referenciador. Como
é feito esse referenciamento em Angular ?
2) Ao fazermos o primeiro teste com ItemComponent, o APP apresentou o
texto "item works!". Porque isso aconteceu ?
3) Para que serve o componente "@Input" utilizado na propriedade "tarefa"
criada para ItemComponent ?
4) Que linha de código representa o exato momento em que uma "tarefa" é
passada para ItemComponent?
5) Para que serve o seguinte trecho dentro da view de ItemComponent:
(change)="tarefa.statusRealizada = !tarefa.statusRealizada;"
6) Permita a utilização da tecla "ENTER" para confirmar o texto da descrição
de uma tarefa, tanto durante a criação de uma nova tarefa, como
também durante a edição de uma tarefa existente.
7) Após criar uma nova tarefa, limpe automaticamente o conteúdo do
campo da descrição.


1R: Por meio de um mecanismo chamado Property Binding 
2R: "item works!" é o texto default que é atribuido a item.component.html quando ele é criado e no momento que testamos isso era tudo que havia dentro do item
3R: @Input permite que você transfira dados de um componente pai para um componente filho, neste caso @Input na propriedade tarefa de ItemComponent faz com que seja passada uma tarefa do App (Pai) para o Item (Filho)
4R: Em app.component.ts dentro de <li> linha <app-item [tarefa]="tarefaDoLoop"></app-item>
5R: Para mudar o status de uma tarefa realizada para não realizada (true para false)

6R:

Para fazer isso devemos usar um evento diretamente no HTML para indicar que a tecla enter foi pressionada, este evento é o (keydown) e para verificar se a tecla apertada foi enter usamos (keydown.enter) e chamamos o método CREATE_tarefa assim como nós fizemos no botão que é usado para criar a tarefa, então é só adicionar na tag input de app app.component.html o evento (keydown.enter)="CREATE_tarefa(campoNovoItem.value)" e no item.component.html, adicionar na tag de input de edição (keydown.enter) seguido do código do botão que confirma a edição, então (keydown.enter)="tarefa.descricao=editedItem.value; emEdicao = false;"

7R: 

Para fazer isso devemos adicionar tanto ao evento (click) do botão que cria a tarefa quanto ao evento (keydown) que cria a tarefa quando enter é pressionado um ";" para indicarmos o término de uma instrução e o começo de outra e adicionar a referencia do input que no caso é campoNovoItem e atribuir a ele o valor "''" então pro input fica (keydown.enter)="CREATE_tarefa(campoNovoItem.value); campoNovoItem.value=''" e para o botão fica (click)="CREATE_tarefa(campoNovoItem.value); campoNovoItem.value=''"
